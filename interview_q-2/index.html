Docker & Kubernetes

Docker imajı ile container arasındaki fark nedir?

İmaj: Uygulamanın çalışması için gerekli dosya ve bağımlılıkların şablonu.

Container: İmajın runtime hali, izole ortamda çalışan uygulama.

Dockerfile optimize etmenin yolları nelerdir?

  • Multi-stage build kullanmak
  • Gereksiz layer'ları azaltmak
  • Minimal base image kullanmak (Alpine)
  • Cache kullanımını optimize etmek

Kubernetes pod nedir?

Pod: Kubernetes'te çalıştırılabilen en küçük deployable birim. İçinde bir veya birden fazla container olabilir.

Deployment ve StatefulSet farkı nedir?

Deployment: Stateless servisler için, pod'lar eşit ve değiştirilebilir

StatefulSet: Stateful servisler için, persistent storage ve ordered deployment sağlar

Kubernetes'de Service tipi ve farkları

  • ClusterIP: Sadece cluster içi erişim
  • NodePort: Dışarıdan belirli port üzerinden erişim
  • LoadBalancer: Cloud provider üzerinden load balancing
  • ExternalName: DNS tabanlı alias

ConfigMap ve Secret farkı nedir?

ConfigMap → plaintext konfig verisi

Secret → şifreli/ hassas veri

Hem pod container'larına mount edilebilir veya env variable olarak kullanılabilir

Liveness ve Readiness probe farkı

Liveness: Pod canlı mı?

Readiness: Pod trafiği alabilir durumda mı?

Docker volume ile bind mount farkı

Volume → Docker tarafından yönetilir, container bağımsız

Bind mount → Host filesystem'e direkt bağlı

Docker networking tipi ve kullanımı

  • bridge → default, container'lar host üzerinde izole
  • host → host network kullanır
  • overlay → swarm veya multi-host network

Kubernetes Horizontal Pod Autoscaler nasıl çalışır?

CPU/memory veya custom metrics bazlı pod sayısını otomatik arttırır veya azaltır

Kafka

Kafka topic, partition ve offset nedir?

Topic → mesajların kategorisi

Partition → paralel işleme birimi

Offset → partition'daki mesaj sırası

Kafka'ya producer ack seviyeleri

  • acks=0 → fire-and-forget
  • acks=1 → leader commit
  • acks=all → leader + replica commit (güvenli)

Kafka consumer group nasıl çalışır?

Partition'lar consumer'lara dağıtılır

Consumer sayısı > partition → fazla consumer idle olur

Kafka'ya exactly-once delivery nasıl sağlanır?

Idempotent producer + transactional producer

Consumer read_committed kullanmalı

Kafka'ya backpressure ve flow control nasıl sağlanır?

Consumer lag monitoring

Reactive Kafka / Reactor kullanarak reactive backpressure

Batch tüketim ve commit ile kontrol

Kafka topic retention nasıl ayarlanır?

retention.ms veya retention.bytes ile

Log segment ve cleanup policy (delete, compact) belirlenir

Kafka vs RabbitMQ farkları

Kafka → event streaming, high throughput, log based

RabbitMQ → queue-based, message acknowledgement, transactional

Kafka consumer lag nasıl izlenir?

Kafka tools veya Prometheus metrics ile

Lag = latest offset - committed offset

WebFlux

WebFlux hangi senaryolarda tercih edilmemeli?

WebFlux, non-blocking IO tabanlıdır. Eğer uygulaman büyük ölçüde CPU-bound işlemler (ör. büyük hesaplamalar, ML algoritmaları) içeriyorsa, WebFlux avantaj sağlamaz. Ayrıca JDBC gibi blokayan kütüphanelerle (ör. klasik Hibernate + MySQL driver) kullanıldığında performans kazancı olmaz, aksine thread starvation yaşanabilir.

Spring WebFlux'te Mono ve Flux farkı nedir?

Mono<T>: 0 veya 1 eleman yayınlar (ör. kullanıcı detayını getir).

Flux<T>: 0-N eleman yayınlar (ör. tüm kullanıcıları getir).

Her ikisi de Publisher'dır, reactive stream standartlarını (Reactive Streams API) uygular.

Reactive WebFlux'te flatMap ve map farkı nedir?

map: Gelen veriyi dönüştürür, Publisher içinde kalır.

flatMap: Asenkron işlemleri zincirlemek için kullanılır, Publisher'ı yayar.

Mono.just("user").map(u -> u.toUpperCase()) // USER

.flatMap(u -> userRepository.findByName(u)) // async DB call

Reactive WebFlux'te backpressure nasıl çözülür?

Backpressure, consumer'ın producer'dan gelen hızla baş edememesi durumudur.

Çözümler:

Flux.onBackpressureBuffer() → buffer ekler

Flux.onBackpressureDrop() → fazla mesajları atar

Flux.limitRate(n) → kontrollü veri çeker

WebFlux ve Servlet tabanlı Spring MVC birlikte kullanılabilir mi?

Evet, Spring Boot 2.x ile hybrid model desteklenir. Ancak aynı controller sınıfı hem WebFlux hem MVC kullanamaz.

Yaygın kullanım:

Public API → WebFlux (yüksek concurrency)

Admin panel / dashboard → Spring MVC

Mikroservis Mimarisi

Mikroservis mimarisinde saga pattern nedir?

Saga, dağıtık transaction yönetiminde kullanılır.

Choreography Saga: Servisler birbirini eventlerle tetikler (Kafka/RabbitMQ).

Orchestration Saga: Merkezi bir orchestrator transaction akışını yönetir.

Avantajı: 2PC (two-phase commit) yerine event-driven compensating transactions ile tutarlılık sağlanır.

Event-driven microservices avantajları nelerdir?

  • Servisler gevşek bağlıdır.
  • Yüksek ölçeklenebilirlik sağlar.
  • Dağıtık transaction yerine eventual consistency ile performans artar.
  • Kafka, RabbitMQ, Pulsar gibi mesajlaşma sistemleri ile doğal entegrasyon yapılır.

API Gateway'de rate limiting nasıl yapılır?

Token Bucket Algorithm veya Leaky Bucket algoritmaları ile yapılır.

Spring Cloud Gateway + Redis kullanarak RequestRateLimiter filter ile IP bazlı rate limiting uygulanabilir.

Microservices ortamında distributed tracing nasıl yapılır?

Spring Cloud Sleuth + Zipkin/Jaeger kullanılır.

Her request'e traceId ve spanId eklenir.

Servisler arası çağrılar loglanır.

Zipkin UI üzerinden request path'leri izlenebilir.

Kafka + Redis + ElasticSearch ile scalable microservice mimarisi nasıl tasarlanır?

Kafka → event streaming / queue

Redis → cache + lightweight pub/sub

ElasticSearch → search / analytics

Spring Cloud → service discovery + config management

Circuit breaker + backpressure + reactive WebFlux → yüksek concurrency ve fault-tolerance sağlanır

Spring Cloud & Microservices

Spring Cloud Eureka nedir?

Resilience4j veya Hystrix ile uygulanır.

Retry, timeout ve bulkhead pattern ile birleştirilebilir.
